# Инструкция по выполнению лабораторной работы №2

## «Литературное программирование: модели SIR и Лотки--Вольтерры»

### Тема и цель

Лабораторная посвящена **литературному программированию** (Literate Programming) на Julia.
Нужно реализовать две классические модели (SIR и Лотки--Вольтерры) сначала в виде обычных Julia-скриптов,
затем **преобразовать их в литературный стиль** и из одного исходного файла
сгенерировать три формата: чистый `.jl`, Jupyter notebook `.ipynb`, Quarto-документацию `.qmd`.

### Порядок действий (по заданию из учебника)

1. Создать рабочий каталог для кода
2. Установить необходимые пакеты
3. **Выполнить предложенный код** (обычные `.jl`-скрипты)
4. **Преобразовать код в литературный стиль**
5. Сгенерировать из литературного кода: чистый код, jupyter notebook, документацию Quarto
6. Выполнить код из jupyter notebook
7. Интегрировать документацию в формате Quarto в отчёт
8. Добавить в литературный код вычисление для набора параметров (анализ чувствительности)
9. Сгенерировать из литературного кода с параметрами: чистый код, jupyter notebook, документацию Quarto
10. Выполнить код из jupyter notebook с параметрами
11. Интегрировать документацию с параметрами в формате Quarto в отчёт

## Шаг 4. Преобразование скриптов в литературный стиль

Это ключевой шаг лабораторной работы. Берём работающие скрипты `sir_ode.jl` и `lv_ode.jl`
и создаём из них литературные версии `sir_ode_literate.jl` и `lv_ode_literate.jl`.

> **Что говорить на видео (вступление к шагу):**
> «Переходим к главному --- преобразованию обычного кода
> в литературный стиль. Идея литературного программирования
> принадлежит Дональду Кнуту, он предложил её в 1984 году.
> Суть в том, что программа --- это не просто набор инструкций для машины,
> а текст, написанный для людей, в котором код объяснён человеческим языком.
>
> В Julia эту концепцию реализует пакет Literate.jl.
> Мы берём наш работающий скрипт и добавляем к нему разметку:
> текстовые пояснения, формулы, таблицы --- всё прямо в комментариях.
> А потом из этого одного файла автоматически генерируются сразу три вещи:
> чистый скрипт, Jupyter notebook и Quarto-документация.
>
> У нас уже есть работающие `sir_ode.jl` и `lv_ode.jl` --- создадим
> их литературные версии.»

### 4.1. Разметка Literate.jl

| Синтаксис    | Назначение                                |
|--------------|-------------------------------------------|
| `# # Заголовок` | Заголовок уровня 1 в Markdown           |
| `# Текст`       | Обычный текстовый параграф              |
| `#jl код`        | Код только для `.jl`-скрипта            |
| `#nb код`        | Код только для `.ipynb`-ноутбука        |
| `#src код`       | Код скрыт из всех выходных форматов     |

> **Что говорить на видео:**
> «Разметка Literate.jl очень простая. Обычные комментарии, начинающиеся с `#`
> и пробела, превращаются в текст Markdown. Два решётки `# #` --- это заголовок.
> Есть три специальных префикса:
> `#jl` --- строка попадёт только в чистый скрипт,
> `#nb` --- только в ноутбук,
> `#src` --- скрыта из всех форматов, это для внутренних нужд.
> Весь остальной код --- общий, он будет и в скрипте, и в ноутбуке, и в документации.»

### 4.2. Литературный скрипт SIR (`scripts/sir_ode_literate.jl`)

Берём код из `sir_ode.jl` и добавляем литературную разметку:

```julia
# # Модель SIR (эпидемиологическая модель)
#
# Стандартная compartmental-модель распространения инфекционного заболевания
# в однородной закрытой популяции без демографических процессов.
#
# ## Теоретические основы
#
# Население делится на три эпидемиологических компартмента:
# - **S** (*Susceptible*) --- восприимчивые к заражению;
# - **I** (*Infectious*) --- инфицированные и заразные;
# - **R** (*Recovered*) --- выздоровевшие и иммунные.
#
# Система ОДУ:
#
# $$\frac{dS}{dt} = -\frac{\beta c I}{N} S$$
# $$\frac{dI}{dt} = \frac{\beta c I}{N} S - \gamma I$$
# $$\frac{dR}{dt} = \gamma I$$
#
# где $N = S + I + R = \text{const}$ --- общая численность популяции.
#
# | Параметр | Значение | Смысл                                       |
# |----------|----------|---------------------------------------------|
# | $\beta$  | 0.05     | вероятность заражения при одном контакте    |
# | $c$      | 10       | среднее число контактов в сутки             |
# | $\gamma$ | 0.25     | скорость выздоровления (1/gamma --- длит. болезни)|
#
# ## Базовое репродуктивное число $R_0$
#
# $$R_0 = \frac{\beta \cdot c}{\gamma}$$
#
# - $R_0 > 1$ --- эпидемия развивается;
# - $R_0 < 1$ --- вспышка затухает.
#
# **Порог коллективного иммунитета**: $1 - 1/R_0$ доля популяции.

# ## Подключение пакетов

using DrWatson
@quickactivate "lab_02_models"
using DifferentialEquations
# ... (остальные using)

# ## Настройка рабочих директорий

#jl script_name = splitext(basename(PROGRAM_FILE))[1]
#nb script_name = "sir_ode"
#jl mkpath(plotsdir(script_name))
#jl mkpath(datadir(script_name))

# ## Определение правой части системы ОДУ
#
# Функция `sir_ode!` вычисляет скорости изменения трёх компартментов.

function sir_ode!(du, u, p, t)
    (S, I, R) = u
    (beta, c, gamma) = p
    N = S + I + R
    @inbounds begin
        du[1] = -beta * c * I / N * S
        du[2] =  beta * c * I / N * S - gamma * I
        du[3] =  gamma * I
    end
    nothing
end

# ... (параметры, решение, визуализация --- тот же код, что в sir_ode.jl)
# ... (каждый блок предваряется текстовым описанием в # комментариях)
# ... (savefig оборачивается в #jl, в ноутбуке графики отрисовываются inline)
```

> **Что говорить на видео:**
> «Вот как выглядит преобразование. Берём наш рабочий скрипт `sir_ode.jl`
> и добавляем литературную разметку.
>
> В начале файла --- теоретическое описание модели SIR прямо в комментариях.
> Используем LaTeX-формулы: когда Literate.jl сгенерирует Markdown-документ,
> эти формулы отрендерятся красиво. А в Julia-скрипте они останутся просто комментариями.
>
> Обратите внимание на директивы `#jl` и `#nb`. В скрипте переменная `PROGRAM_FILE`
> содержит имя запущенного файла --- мы из неё извлекаем название.
> А в ноутбуке `PROGRAM_FILE` не существует, поэтому задаём имя вручную.
> `savefig` тоже обёрнут в `#jl` --- при запуске как скрипт графики сохраняются в файлы,
> а в ноутбуке они отрисовываются inline.
>
> Сам код модели остаётся **тем же самым** --- мы просто добавили текстовые пояснения
> и разграничили поведение для разных форматов.»

### 6.2. Выполнение ноутбуков через nbconvert

Из терминала, находясь в каталоге `lab_02_models`:

```bash
jupyter nbconvert \
    --to notebook --execute \
    --ExecutePreprocessor.kernel_name=julia-1.12 \
    --ExecutePreprocessor.timeout=600 \
    --inplace notebooks/lv_ode.ipynb
```

```bash
jupyter nbconvert \
    --to notebook --execute \
    --ExecutePreprocessor.kernel_name=julia-1.12 \
    --ExecutePreprocessor.timeout=600 \
    --inplace notebooks/sir_ode.ipynb
```

Параметры:
- `--to notebook --execute` --- выполнить ноутбук и записать результаты
- `--ExecutePreprocessor.kernel_name=julia-1.12` --- явно указать ядро Julia
- `--ExecutePreprocessor.timeout=600` --- таймаут 10 минут (компиляция пакетов долгая)
- `--inplace` --- записать результаты обратно в тот же файл

> **Что говорить на видео:**
> «Сгенерированные ноутбуки пока пустые --- в них есть код, но он не выполнен.

````markdown
```julia
function sir_ode!(du, u, p, t)
    S, I, R = u
    beta, c, gamma = p
    N = S + I + R
    du[1] = -(beta * c / N) * I * S
    du[2] = (beta * c / N) * I * S - gamma * I
    du[3] = gamma * I
end
````

## Ключевые ожидаемые результаты

### Модель SIR (beta=0.05, c=10, gamma=0.25)

- R0 = 2.0 --- эпидемия развивается
- Пик заражённых: I_max ~ 154.8 чел. на день ~19.1
- Итоговый охват: ~77.6% популяции
- Порог коллективного иммунитета: 50%
- При R0 < 1 эпидемия не развивается; при R0 = 4 переболевают > 95%

### Модель Лотки--Вольтерры (alpha=0.1, beta=0.02, delta=0.01, gamma=0.3)

- Нетривиальное равновесие: x* = gamma/delta = 30, y* = alpha/beta = 5
- Теоретический период: T = 2*pi / sqrt(alpha*gamma) ~ 36.3
- Замкнутые орбиты на фазовом портрете (консервативная система)
- Доминирующая частота FFT совпадает с теоретической оценкой
- Хищники отстают от жертв по фазе (классический сдвиг фаз)

### Анализ чувствительности

**SIR:** при увеличении R0 от 0.5 до 4.0 --- нелинейный рост охвата и пика;
резкое изменение вблизи порога R0 = 1.

**LV:** изменение параметров смещает равновесную точку и амплитуду колебаний,
но замкнутые орбиты сохраняются.
